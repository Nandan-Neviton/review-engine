# US-106

## Title
Implement Secure Session-Based Authentication Flow

## Business Context

The system should securely authenticate users and manage active sessions across login flows.

## Requirements

- Create login service
- Create session service
- Validate email format
- Validate password strength (min 8 chars, uppercase, numeric, special character)
- Reject empty credentials
- Generate cryptographically secure session identifiers (crypto.randomBytes or crypto.randomUUID)
- Session expiration should be configurable via SESSION_TTL_MINUTES env variable with safe bounds (5-1440 min, default 30)
- Return structured API responses using a shared response helper
- Follow service-layer architecture (routes must not directly access sessions)
- Reuse validation helpers where possible
- Avoid console.log statements
- Avoid TODO/FIXME comments

## Security Requirements

- No hardcoded secrets
- No sensitive logging
- Auth/session modules require governance review
- Session identifiers must use Node.js crypto module — Math.random() is strictly prohibited
- Session expiration must be configurable through environment variables with bounds validation
- Password validation must enforce complexity (length, uppercase, numeric, special character)
- Shared response helper must be used for all auth responses — no duplicated response logic

## Architecture Compliance

- Modules must reside under src/auth/ and src/utils/
- Import paths must resolve correctly — no broken require() calls
- session-validator.js must import from ./session-manager (not ../session/session-validator)
- Hybrid auth strategy: session-manager handles pre-auth sessions, JWT handles API access
- Architecture document comments must clarify the JWT vs session relationship
- All modules must be independently testable via dependency injection

## Acceptance Criteria

- login-service.js correctly validates email and password before proceeding
- session-manager.js creates sessions with cryptographically secure IDs
- session-manager.js parses SESSION_TTL_MINUTES with parseInt and bounds validation
- session-validator.js imports from the correct relative path
- All auth services use response-helper.js — no createResponse/createApiResponse duplication
- No Math.random() usage anywhere in auth modules
